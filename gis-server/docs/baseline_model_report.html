Executive Summary

Objective

Establish a performance baseline for Bangkok property price prediction using LightGBM with heavy spatial feature engineering. This model serves as a benchmark before deploying graph neural networks (HGT).

Key Finding

The baseline achieves R² = 0.80 with spatial cross-validation, indicating strong predictive power. Adding Hex2Vec embeddings improves to R² = 0.82, demonstrating value of learned spatial representations.

Evaluation Methodology

Spatial Cross-Validation (GroupKFold)

We use 5-fold GroupKFold by district to test generalization to unseen neighborhoods. This prevents data leakage from spatially correlated properties.

Fold 1: Train on 39 districts, Test on หนองจอก, ราชเทวี
Fold 2: Train on 31 districts, Test on บางกะปิ, ประเวศ, ...
Fold 3: Train on 31 districts, Test on ลาดกระบัง, ลาดพร้าว, ...
Fold 4: Train on 31 districts, Test on มีนบุรี, สายไหม, ...
Fold 5: Train on 32 districts, Test on หนองแขม, สะพานสูง, ...

Why NOT Random Split?

Random 80/20 splits cause spatial leakage: training on neighbors of test properties artificially inflates metrics. Our spatial split tests true generalization.

⚠️ Warning: Random split R² would likely be ~0.90+ but would fail on real unseen neighborhoods. Spatial CV is harder but more realistic.

  • • POI counts (school, hospital, mall...)
  • • Transit stops in hexagon
  • • Property count (development density)
  • • Flood risk (district-level)

4 Hex2Vec Embeddings (+64)

64-dimensional learned spatial embeddings from Skip-gram model trained on H3 cell co-occurrences. Captures latent neighborhood similarity beyond explicit POI counts.

Target Transformation

Log Transformation: log_price = log1p(total_price)

Real estate prices follow a power-law distribution. Log transform normalizes the distribution and prevents the model from over-focusing on luxury properties.
